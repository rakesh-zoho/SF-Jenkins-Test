# Agent Context — Quick Reference
> Include in your Copilot Chat prompt when starting any agent session.

## Project Layout
| Path | Purpose |
|---|---|
| `tasks/*.md` | **You write** — plain English flow description |
| `specs/*-plan.md` | **Planner writes** — detailed step-by-step plan |
| `tests/*.spec.js` | **Generator writes** — executable Playwright tests |
| `tests/seed.spec.js` | Auth fixture — agents copy this into every generated test |
| `utils/sf-helpers.js` | waitForSFLoad, switchToAllRecords, navigateToApp |
| `utils/reporter-utils.js` | captureScreenshot, sfStep, setAllureMeta |
| `utils/locator-utils.js` | fillField, selectPicklist, fillLookup, getDatePlusDays, uniqueName |
| `memory/framework-memory.md` | All agent rules and SF patterns |
| `memory/sf-selectors.md` | Full SF locator reference |
| `reports/.auth-state.json` | Saved SF auth (generated by globalSetup) |

## Planner Agent — What to Do
1. Read `tasks/[name].md` carefully
2. Read `memory/framework-memory.md` for rules
3. Read `memory/sf-selectors.md` for known selectors
4. Open Salesforce via MCP browser tools (`browser_navigate`)
5. Explore each UI step — take screenshots
6. Write detailed plan to `specs/[name]-plan.md`
7. Plan must include: exact locators, wait conditions, assertions, screenshot points

## Generator Agent — What to Produce
1. Read `specs/[name]-plan.md`
2. Read `tests/seed.spec.js` for the auth + fixture pattern
3. Read `utils/reporter-utils.js` and `utils/locator-utils.js` for available helpers
4. Generate `tests/[name].spec.js` with:
   - `test.use({ storageState: './reports/.auth-state.json' })`
   - `allure.epic/feature/story/severity` in beforeEach
   - `sfStep(name, page, fn)` wrappers for every major action
   - `captureScreenshot(page, 'name')` after every step
   - `expect()` assertions after every action
   - `afterEach` that screenshots on failure
   - Imports from `../utils/sf-helpers.js`, `../utils/reporter-utils.js`, `../utils/locator-utils.js`

## Healer Agent — How to Fix
1. Run `npx playwright test tests/[name].spec.js --config=config/playwright.config.js`
2. For each failure:
   - Read the error message and stack trace
   - Check `memory/sf-selectors.md` for alternative locators
   - Open Salesforce via MCP browser tools to inspect the element if needed
   - Patch the broken locator in the spec file
   - Add comment: `// HEALED: [reason for change]`
3. Save a `.bak` backup before patching
4. Re-run and repeat until all tests pass
5. Update `memory/framework-memory.md` with any new lessons learned
